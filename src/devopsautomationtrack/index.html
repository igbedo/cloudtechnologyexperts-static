Program Structure

DevOps & Automation Track Curriculum

The curriculum starts with Linux, Git, and scripting fundamentals, then moves into CI/CD, containers, Kubernetes basics, and Infrastructure as Code, ending with a real pipeline project.

Module 1
Linux, Shell & Git Foundations
  • Linux recap for DevOps: services, processes, permissions
  • Command line productivity & shell navigation
  • Git basics, branching, merging & pull requests
Module 2
Scripting & Automation Fundamentals
  • Bash scripting for routine tasks & automation
  • Intro to Python for DevOps (optional/essentials)
  • Working with APIs, CLI tools, and JSON/YAML configs
Module 3
CI/CD Concepts & Pipelines
  • What is CI/CD and why it matters
  • Build pipelines with GitHub Actions / GitLab CI / Jenkins (choose stack)
  • Automated testing, artifact storage & deployment triggers
Module 4
Containers & Docker
  • Container fundamentals and images vs. containers
  • Dockerfiles, multi-stage builds & registries
  • Running and debugging containerized applications
Module 5
Kubernetes Basics for DevOps
  • Core concepts: Pods, Deployments, Services, ConfigMaps
  • Using kubectl, YAML manifests & namespaces
  • Rolling updates and basic troubleshooting
Module 6
Infrastructure as Code & Observability
  • IaC with Terraform (or similar): providers, resources, state
  • Provisioning cloud infrastructure for pipelines
  • Monitoring builds & applications; logs & alerts
Capstone Project
End-to-End CI/CD Pipeline for a Cloud Application

Build a full CI/CD pipeline that takes a simple application from source code to production: containerize it with Docker, provision infrastructure with IaC, deploy to a cloud environment, and set up basic monitoring. Document your pipeline and present it as a portfolio project.
